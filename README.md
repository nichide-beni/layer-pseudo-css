# 擬似要素とcss layerを使った戦略的なcss構築

## :isと:where

:is, :whereともに動きはほぼ同じで、引数に指定したセレクタに該当したもの全てにスタイルを適用する。  
両者の違いは、引数の結果の詳細度が:isは引数で最も高い詳細度に引っ張られ、:whereでは-になる。  

### :is, :has両者の使い分け

- :isは通常使用をするとき
- :whereはライブラリや、上書きを前提として使用するとき

### 書き方

:is(セレクター1, セレクター2)   {}
:where(セレクター1, セレクター2) {}

### :hasによる別のセレクタを使った記述

これまで実現できなかった「特定の子要素を兄弟を持つ要素」を指定できるようになった。  
つまり、後で記述する要素によって先に記述された要素にスタイルを当てることができる。  

### sample01 擬似セレクタの応用

擬似セレクタの:is, :where, :hasをそれぞれ使ったサンプル。  
isやwhereを使ってフォーム部品や見出しなど共通で設定するようなスタイルを設定し、hasではチェック状態や親子関係の有無など柔軟な状態判定が可能。  

- whereは上書きしやすいようにベースのスタイルとして
- isはセレクタとして使用箇所を想定した形式で使用(通常はこちらを採用する可能性が高い)
- hasは汎用性が高く、JavaScriptで判定していたように状態によってスタイルを切り分ける。

## レイヤーによる優先順位の制御

cssの@layerでレイヤー順を指定すると、詳細度をよりも優先してスタイルを適用することができる。  
上書きされやすい共通スタイルなどを上層のレイヤーで宣言し、個別のコンテンツスタイルを後から宣言する。  
ライブラリをそのまま読み込むとレイヤー外の強いスタイルになる可能性があるため、cssでimportするなどの工夫が必要。  

### レイヤーの優先順位

@layer layer1, layer2, layer3, ..., と宣言した場合、後で宣言したlayer3で宣言したスタイルが適用される。  
あくまでスタイル単位の適用になるため、layer1でのみ宣言したスタイルはそのまま適用される。  

#### importantの優先順位

importantの場合はレイヤーの宣言順(先が優先)される仕様になっているため注意が必要。  
上書きをしたい場合は、レイヤー外に宣言する必要がある。  
レイヤーの詳細: https://developer.mozilla.org/en-US/docs/Web/CSS/Reference/At-rules/@layer  

#### レイヤーの優先順位まとめ

- 通常のスタイル(優先順)
 - レイヤーの外で宣言されたスタイル
 - レイヤー宣言の逆順、後で宣言したレイヤーが優先
- important指定によるスタイル(優先度)
 - レイヤーの外で宣言されたスタイル
  - レイヤー宣言順、先に宣言したレイヤーが優先
